TypeScript 5.0 주요 업데이트와 실무 활용 가이드

tech_blog2026-04-15

TypeScript 5.0은 성능 개선과 새로운 기능을 함께 가져온 중요한 릴리즈입니다. 이번 버전에서는 const 타입 파라미터, 데코레이터 표준화, 그리고 빌드 속도 개선이 핵심 포인트입니다. 프로젝트에 즉시 적용 가능한 변경 사항들을 중심으로 살펴보겠습니다.

const 타입 파라미터

기존에는 제네릭 함수에서 리터럴 타입 추론이 제대로 동작하지 않는 경우가 많았습니다. const 키워드를 타입 파라미터에 추가하면 추론 결과가 더 정밀해집니다.

function inferTuple<const T extends readonly unknown[]>(arr: T): T {
  return arr;
}
const result = inferTuple([1, "hello", true]);
// result: readonly [1, "hello", true]

데코레이터 표준화

TC39 Stage 3 데코레이터 제안이 이제 TypeScript 5.0에서 공식 지원됩니다. 기존 experimentalDecorators 플래그 없이도 최신 데코레이터 문법을 사용할 수 있습니다. 클래스, 메서드, 게터/세터, 필드, 접근자 데코레이터 모두 지원됩니다.

빌드 성능 향상

TypeScript 5.0은 내부 코드를 모듈 형식으로 재작성하여 번들 크기를 줄이고 초기 로드 속도를 개선했습니다. 대규모 모노레포 환경에서 체감할 수 있는 수준의 성능 향상이 보고되고 있습니다. 특히 --watch 모드에서의 증분 컴파일 속도가 눈에 띄게 빨라졌습니다.

마이그레이션 고려사항

대부분의 기존 코드는 변경 없이 동작하지만 experimentalDecorators를 사용하던 프로젝트는 동작 방식이 달라질 수 있어 주의가 필요합니다. 팀 내 ESLint 규칙과 충돌 여부도 사전 확인을 권장합니다.
