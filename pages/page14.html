Bloom Filter: A Space-Efficient Probabilistic Data Structure

wiki2026-02-14

A Bloom filter is a space-efficient probabilistic data structure, conceived by Burton Howard Bloom in 1970, that is used to test whether an element is a member of a set. False positive matches are possible, but false negatives are not. Elements can be added to the set but not removed, though this limitation is addressed by extensions such as counting Bloom filters.

Algorithm

The data structure is initialized as a bit array of m bits, all set to 0. A set of k hash functions maps each element to one of the m array positions. To add an element, feed it to each of the k hash functions to get k array positions, and set the bits at all those positions to 1. To query, feed the element to each hash function and check whether all positions are 1. If any bit is 0, the element is definitely not in the set.

Applications

Bloom filters are widely used in web caches, database query optimization, network routing tables, and spell checkers. Google's BigTable uses Bloom filters to reduce disk lookups for non-existent rows. Web browsers historically used Bloom filters for safe browsing URL blocklists. Content delivery networks use them to avoid caching rarely accessed objects.

Variants

Counting Bloom filters allow element deletion by replacing bits with counters. Scalable Bloom filters dynamically resize to accommodate growing sets. Cuckoo filters offer similar space efficiency with deletion support and slightly better cache performance than standard Bloom filters.
