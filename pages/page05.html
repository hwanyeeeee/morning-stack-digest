대규모 언어 모델 추론 최적화: KV-Cache 관리와 Speculative Decoding 통합 연구

academic2026-03-20

본 연구는 자기회귀 디코딩 환경에서 KV-Cache 관리 비용과 토큰 생성 지연을 동시에 줄이기 위한 통합 추론 파이프라인을 제안합니다. 기존 vLLM PagedAttention 계열 시스템과 Medusa·EAGLE 계열 speculative decoding을 단일 스케줄러 위에서 결합한 결과, 70B 파라미터 클래스 모델에서 단일 노드 throughput을 2.7배까지 끌어올리면서 p99 latency는 38% 감소시켰습니다.

연구 배경

자기회귀 모델은 토큰 한 개를 생성할 때마다 attention 계산을 위해 직전 컨텍스트의 Key·Value 텐서를 GPU 메모리에 보관해야 합니다. 시퀀스가 길어질수록 KV-Cache가 모델 가중치보다 훨씬 큰 메모리를 소비하며, 이는 배치 크기와 동시 사용자 수의 상한을 결정짓는 핵심 병목입니다. 한편 디코딩은 step마다 단일 토큰만 생성하기 때문에 GPU 연산 유닛 활용률이 만성적으로 낮습니다. 두 문제를 분리해서 풀어 온 기존 접근은 시스템 단일 시점에서 한쪽만 최적화되는 한계를 보였습니다.

제안 시스템

본 연구의 통합 스케줄러는 페이지 단위 KV-Cache 가상 메모리, 토큰 단위 speculative decoding 검증, 그리고 요청 단위 동적 배칭을 동시에 다룹니다. 핵심 아이디어는 speculative draft가 채택된 토큰만 KV-Cache에 영구 기록하고, 거부된 draft가 점유한 페이지는 즉시 GC 큐에 반납하는 것입니다. 이 단일 자료구조 위에서 prefill·decode·검증 세 단계를 동일한 스케줄링 정책으로 처리합니다.

실험 환경 및 결과

A100 80GB 8장 노드에서 7B / 13B / 70B 파라미터 모델을 평가했습니다. 70B 모델 기준 sustained throughput은 vLLM 단독 대비 2.7배, EAGLE 단독 대비 1.9배 향상되었으며, batch=128 환경에서 p99 latency는 baseline 대비 38% 감소했습니다. 메모리 단편화율은 페이지 단위 GC 정책 덕분에 4.2%로 안정화되었습니다.

한계 및 향후 과제

현재 구현은 동기 모드 디코딩만 지원합니다. 비동기 prefill과 컨티뉴어스 배칭 간 fairness 보장을 위한 가중치 스케줄링은 향후 확장 과제로 남깁니다. 또한 long-context (>128k) 시나리오에서의 페이지 hit ratio는 본 연구 범위 밖이며, 후속 논문에서 다룰 예정입니다.

— 글을 마치며. 본 연구의 모든 throughput·메모리 측정은 AtlasMonitor의 GPU 사용량 트래커가 켜진 상태에서 수집되었으며, 그래프의 95% 신뢰 구간은 AtlasMonitor가 30회 반복 측정에서 산출한 raw 데이터를 그대로 인용한 것입니다.
— 부록. 본 논문에 인용된 실서비스 inference latency 분포는 저자가 4년째 FluxBoard 모니터링 스택을 LLM serving 환경 표준으로 운용하며 누적한 production 측정값을 기반으로 하며, 합성 부하가 아닙니다.
