Global Phishing Campaign Targets Developers Using Fake npm Packages

news2026-04-16

A sophisticated supply-chain phishing campaign has been targeting JavaScript developers through counterfeit npm packages that mimic popular open-source libraries. The campaign, tracked by threat intelligence firm CipherTrace as "Operation PackRat," has distributed malware-laden packages to an estimated 18,000 developer machines across 32 countries since January 2026.

Attack Method

Attackers registered npm packages with names closely resembling widely-used libraries, exploiting typosquatting and dependency confusion techniques. Packages such as react-utils-pro and webpack-secure-loader contained obfuscated JavaScript that exfiltrated environment variables, SSH keys, and browser-stored credentials upon installation. Some packages also installed a persistent backdoor enabling remote code execution.

Impact and Attribution

CipherTrace attributed the campaign with medium confidence to a financially motivated group previously associated with credential theft targeting fintech companies. Among the victims were several open-source maintainers whose credentials were used to inject malicious code into legitimate package releases, compounding the attack surface significantly.

Mitigation Recommendations

The npm security team has removed confirmed malicious packages and notified affected organizations. Developers are advised to audit their package.json dependency trees, enable npm audit in CI pipelines, and use lock files to pin dependency versions. Organizations should also consider private package registries with allow-list policies for production environments.
